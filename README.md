# commitcraft

AI-assisted [Conventional Commit](https://www.conventionalcommits.org/) message
generator. Stage your changes, run `commitcraft`, get a clean commit message
back — written by Claude when you have an API key, or by a deterministic
offline classifier when you don't.

- **Reads only.** commitcraft never touches your working tree or staged
  changes. It only writes when you explicitly pass `--apply`, and even then
  it only runs `git commit -m "<message>"`.
- **AI-assisted by default.** With `ANTHROPIC_API_KEY` set, the staged diff
  is sent to Claude, which returns a single Conventional Commit message
  (type, optional scope, imperative summary, optional bullet body).
- **Fails gracefully.** No API key, no `anthropic` package, or a network
  error — commitcraft falls back to a rule-based classifier that looks at
  changed file paths and insertion/deletion counts to pick a commit type
  (`feat`, `fix`, `docs`, `test`, `chore`, `refactor`) and scope.

## Install

```bash
pip install -r requirements.txt
pip install -e .
```

## Usage

```bash
git add .

# Preview a message (AI if ANTHROPIC_API_KEY is set, else offline)
commitcraft

# Preview and commit in one step
commitcraft --apply

# Force the offline heuristic even with a key set
commitcraft --no-ai
```

Example output:

```
--- generated by Claude ---
feat(auth): add OAuth login flow

- Add login route and callback handler
- Store refresh tokens in secure cookie
-----------------------------
Run with --apply to commit with this message.
```

## How it works

1. `gitutil.py` shells out to `git diff --cached` to read the staged diff,
   changed file list, and insertion/deletion counts. Raises a clear error
   if you're not in a repo or have nothing staged.
2. `generator.py` sends that diff to Claude with a system prompt that
   constrains the reply to a single Conventional Commit message, then
   parses the response into a structured `CommitMessage`.
3. `heuristics.py` provides the offline path: pattern-matches file paths
   (tests/, docs/, config files) and diff shape (net additions vs. net
   deletions) to guess a commit type, and infers a scope when every
   changed file shares one top-level directory.
4. `cli.py` wires it together behind a `commitcraft` command built with
   `click`.

## Development

```bash
pip install -r requirements.txt
pytest
```

All tests run against synthetic `StagedChange` objects and mocked/absent
API keys — no real git repo or network access required.

## Project layout

```
src/commitcraft/
  gitutil.py     # subprocess wrapper for reading staged diffs (and committing)
  heuristics.py  # offline commit-type classifier + message builder
  generator.py   # Claude API call, prompt, and response parsing
  cli.py         # click-based command line entry point
tests/           # pytest suite, no network or real git repo required
```
